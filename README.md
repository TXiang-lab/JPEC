# JPEC <img src="https://img.shields.io/badge/Issues-%2B-brightgreen.svg" /><img src="https://img.shields.io/badge/license-GPL3.0-blue.svg" />
JPEC is a Julia tool for pedigree error correction in animal and plant breeding.

It compares phased genotype haplotypes between offspring and candidate parents, then reports the most likely sire or dam. It can be used to check recorded pedigree information, find likely parentage mistakes, and generate a corrected pedigree.

## What JPEC Needs

Every JPEC run needs two files:

1. A phased VCF genotype file.
2. An LD block file, usually generated by PLINK.

A pedigree file is optional but strongly recommended. If no pedigree is provided, you must provide both an offspring list and a candidate parent list.

JPEC can be used in three common ways:

| Goal                                                        | Required files                                 |
| ----------------------------------------------------------- | ---------------------------------------------- |
| Correct all eligible records in a pedigree                  | VCF + LD blocks + pedigree                     |
| Correct selected offspring from a pedigree                  | VCF + LD blocks + pedigree + offspring list    |
| Match selected offspring against selected candidate parents | VCF + LD blocks + offspring list + parent list |

Only individuals present in the VCF can be analyzed.

## Input Files

### 1. Pedigree File

The pedigree file should have no header. It must contain 4 columns, or 5 columns if sex information is available.

Required columns:

| Column | Meaning                  | Example           |
| ------ | ------------------------ | ----------------- |
| 1      | Individual ID            | `76080`           |
| 2      | Sire ID                  | `63032`           |
| 3      | Dam ID                   | `67598`           |
| 4      | Generation or birth date | `3` or `20231001` |

Optional column:

| Column | Meaning | Allowed values               |
| ------ | ------- | ---------------------------- |
| 5      | Sex     | `M` for male, `F` for female |

Use `0` for unknown sire or dam.

Example:

```text
54903 0 0 1
55402 0 0 1
55685 0 0 1
76080 63032 67598 3
76092 64806 69141 3
```

If your pedigree does not contain generation records, we recommend first ordering the pedigree from older to younger individuals. One option is the `trace_pedigree` function in the R package [blupADC](https://github.com/TXiang-lab/blupADC).

### 2. Phased VCF File

The genotype file must be phased VCF. Genotypes should use phased separators such as `0|0`, `0|1`, `1|0`, and `1|1`.

Example:

```text
##fileformat=VCFv4.2
#CHROM  POS     ID      REF     ALT     QUAL    FILTER  INFO    FORMAT  55402   55685
1       6260    M2      T       A       .       PASS    .       GT      0|0     0|1
1       152890  M17     A       T       .       PASS    .       GT      1|0     1|1
```

You can phase genotypes with Beagle 5.2:

```bash
java -Xmx50g -jar beagle.5.2.jar \
  gt=genotype_file.vcf \
  out=haplotype \
  impute=true \
  nthreads=10
```

Beagle 5.2 download page:

https://faculty.washington.edu/browning/beagle/b5_2.html#download

### 3. LD Block File

The LD block file should use the PLINK `.blocks.det` format. JPEC uses the first three columns: chromosome, block start position, and block end position.

Example:

```text
CHR          BP1          BP2           KB  NSNPS SNPS
1          282580       431960      149.381      4 M30|M33|M37|M46
1         3161360      3358350      196.991     12 M344|M347|M349
1         4505080      4545150       40.071      4 M499|M500|M501
```

You can generate LD blocks with PLINK 1.9:

```bash
plink --vcf haplotype.vcf --blocks no-pheno-req --blocks-max-kb 200
```

PLINK download page:

https://www.cog-genomics.org/plink/

### 4. Offspring List

Optional. Use one ID per line. No header is required.

```text
76080
76092
76271
```

### 5. Candidate Parent List

Optional. Use one ID per line. No header is required.

```text
63032
64806
67598
69141
```

<a id="missing-values"></a>

## Missing Values

Based on the current code, please use `0` for missing values in the pedigree.

| File / field                  | Accepted missing value | Notes                                                        |
| ----------------------------- | ---------------------- | ------------------------------------------------------------ |
| Pedigree sire and dam columns | `0`                    | This is the recommended and supported code for unknown parents. |
| Pedigree birth-date column    | `0`                    | When birth dates are used, `0` is internally treated as `20010101` for date parsing. |
| Offspring list / parent list  | none                   | Keep one valid ID per line. Do not include `NA` rows.        |
| Main phased VCF genotype      | not recommended        | The main VCF reader expects phased genotypes such as `0|0`, `0|1`, `1|0`, and `1|1`. Missing VCF genotypes such as `./.` are not handled as true missing values in the main matching code. |
| chrX VCF for swap checking    | `.` in genotype        | The chrX swap-check reader treats genotypes containing `.` as missing. |

Do not use `NA` in the pedigree. In the current code, `NA` is treated as a normal string, not as a missing value. For example, `NA` in sire or dam columns is not equivalent to `0`, and `NA` in the birth-date column can cause date parsing to fail.

<a id="how-to-run"></a>

## How To Run

The examples below use the compiled JPEC binary:

```bash
../bin/JPEC [options] --julia-args --threads=2
```

Options before `--julia-args` are JPEC options. Options after `--julia-args` are Julia runtime options.

### Example 1: Correct Sires In A Pedigree

```bash
../bin/JPEC \
  --blocks example_data/plink.blocks.det \
  --vcf example_data/haplotype.vcf \
  --ped example_data/ped.txt \
  --findex 1 \
  --o result \
  --julia-args --threads=2
```

### Example 2: Correct Both Sires And Dams

```bash
../bin/JPEC \
  --blocks example_data/plink.blocks.det \
  --vcf example_data/haplotype.vcf \
  --ped example_data/ped.txt \
  --findex 3 \
  --o result \
  --julia-args --threads=2
```

### Example 3: Match Selected Offspring Against Selected Parents

```bash
../bin/JPEC \
  --blocks example_data/plink.blocks.det \
  --vcf example_data/haplotype.vcf \
  --offspring example_data/off_id.txt \
  --parents example_data/par_id.txt \
  --findex 1 \
  --score \
  --o result \
  --julia-args --threads=2
```

<a id="command-line-options"></a>

## Command-Line Options

### Required Options

| Option          | Meaning                                    |
| --------------- | ------------------------------------------ |
| `--blocks FILE` | LD block file, usually `plink.blocks.det`. |
| `--vcf FILE`    | Phased VCF genotype file.                  |

### Parent Search Options

| Option       | Meaning                         | Default |
| ------------ | ------------------------------- | ------- |
| `--findex 1` | Search for sires only.          |         |
| `--findex 2` | Search for dams only.           |         |
| `--findex 3` | Search for both sires and dams. | yes     |

If `--findex` is not provided, JPEC searches both sire and dam.

### Optional Input Options

| Option             | Meaning                            |
| ------------------ | ---------------------------------- |
| `--ped FILE`       | Pedigree file with 4 or 5 columns. |
| `--offspring FILE` | Target offspring list.             |
| `--parents FILE`   | Candidate parent list.             |

### Filtering Options

| Option         | Meaning                                                      | Default   |
| -------------- | ------------------------------------------------------------ | --------- |
| `--gen NUMBER` | Minimum generation gap between parent and offspring. Used when pedigree column 4 is generation. | `1`       |
| `--day NUMBER` | Minimum birth-date interval in days. Used when pedigree column 4 is birth date. | automatic |

When `--day` is not provided and the fourth pedigree column looks like birth dates, JPEC estimates the minimum parent-offspring day interval from the recorded pedigree.

### Output And Runtime Options

| Option                     | Meaning                                                      | Default           |
| -------------------------- | ------------------------------------------------------------ | ----------------- |
| `--o DIR`                  | Output directory. JPEC creates it if needed.                 | current directory |
| `--score`                  | Write the full offspring-by-parent score matrix.             | off               |
| `--score-threshold NUMBER` | Minimum score required before JPEC accepts a matched parent for correction/reporting. | `0.9`             |
| `--noMC`                   | Skip MC-block adjustment.                                    | off               |
| `--nGC NUMBER`             | Control how often garbage collection is triggered during MC-score calculation. Usually leave this unset. | automatic         |
| `--julia-args --threads=N` | Run with `N` Julia threads when using the compiled binary.   | Julia default     |

<a id="output-files"></a>

## Output Files

The output files are written to the directory given by `--o`.

### Corrected Parent Records

For sire correction:

```text
JPEC_vs_Original_Sire.txt
```

For dam correction:

```text
JPEC_vs_Original_Dam.txt
```

These files contain records where JPEC found a different parent than the one in the pedigree.

Example:

```text
offspring Original JPEC
76092 64806 63032
76271 64712 62673
76475 65120 69518
```

### Corrected Pedigree

```text
JPEC_ped.txt
```

This file contains the updated pedigree after correction.

Example:

```text
76080 63032 67598 3
76092 63032 69141 3
76269 62673 65612 3
```

### Matched Parents Without A Pedigree

If no pedigree is provided, JPEC writes:

```text
JPEC_matched_Sire.txt
JPEC_matched_Dam.txt
```

depending on the `--findex` setting.

Example:

```text
offspring matched_parents best_score passed_threshold
76092 63032 0.981 true
76271 0 0.842 false
```

If the best score is below `--score-threshold`, `matched_parents` is reported as `0`.

### Offspring And Parent Lists Used By JPEC

JPEC writes the final offspring and candidate parent sets used for matching:

```text
JPEC_Sire_off_id.txt
JPEC_Sire_par_id.txt
JPEC_Dam_off_id.txt
JPEC_Dam_par_id.txt
```

The sire files are produced when sire matching is run. The dam files are produced when dam matching is run.

### Score Matrix

If `--score` is used, JPEC writes a score matrix:

```text
JPEC_Sire_socred_matrix.txt
JPEC_Dam_socred_matrix.txt
```

Note: the current code writes `socred` in the file name. This looks like a typo, but it is the current output name.

Rows are offspring. Columns are candidate parents.

Example:

```text
Offspring_Parents  61202  58769  60903
62673              0.966  0.407  0.380
62981              0.234  0.977  0.299
```

<a id="optional-sire-dam-swap-check"></a>

## Optional Sire-Dam Swap Check

The compiled JPEC command can also check whether sire and dam IDs may have been placed in the wrong columns.

Swap checking can use pedigree evidence, sex-chromosome evidence, or both.

### Swap-Check Modes

Use `--sex-swap MODE`.

| Mode            | Meaning                                                      |
| --------------- | ------------------------------------------------------------ |
| `pedigree`      | Use pedigree evidence only.                                  |
| `chrX`          | Use sex-chromosome evidence only.                            |
| `pedigree_chrX` | Use both; pedigree evidence has priority if they conflict.   |
| `chrX_pedigree` | Use both; sex-chromosome evidence has priority if they conflict. |

### Example: Pedigree-Based Swap Check

```bash
../bin/JPEC \
  --blocks example_data/plink.blocks.det \
  --vcf example_data/haplotype.vcf \
  --ped example_data/ped.txt \
  --findex 3 \
  --sex-swap pedigree \
  --swap-threshold 0.90 \
  --swap-min-count 3 \
  --o result \
  --julia-args --threads=2
```

### Example: Pedigree And chrX Swap Check

```bash
../bin/JPEC \
  --blocks example_data/plink.blocks.det \
  --vcf example_data/haplotype.vcf \
  --ped example_data/ped.txt \
  --findex 3 \
  --sex-swap pedigree_chrX \
  --chrX-vcf-file example_data/chrX.vcf \
  --chrX-names X,chrX \
  --male-het-range 0.0,0.05 \
  --female-het-range 0.20,1.0 \
  --o result \
  --julia-args --threads=2
```

### Swap-Check Options

| Option                       | Meaning                                                      | Default    |
| ---------------------------- | ------------------------------------------------------------ | ---------- |
| `--sex-swap MODE`            | Turn on sire-dam swap checking.                              | off        |
| `--swap-threshold NUMBER`    | Minimum sire-column or dam-column proportion used by pedigree-based checking. | `0.90`     |
| `--swap-min-count NUMBER`    | Minimum number of pedigree occurrences before an ID is evaluated. | `3`        |
| `--chrX-vcf-file FILE`       | VCF file containing sex-chromosome markers. Required for `chrX`, `pedigree_chrX`, and `chrX_pedigree` modes. | none       |
| `--chrX-names NAMES`         | Sex-chromosome labels in the VCF, separated by commas.       | `X,chrX`   |
| `--male-het-range MIN,MAX`   | Expected heterozygosity range for males.                     | `0.0,0.05` |
| `--female-het-range MIN,MAX` | Expected heterozygosity range for females.                   | `0.20,1.0` |

For species where the sex-chromosome pattern is reversed, swap the male and female heterozygosity ranges. For example, chicken chrZ may use:

```text
--male-het-range 0.20,1.0
--female-het-range 0.0,0.05
```

<a id="troubleshooting"></a>

## Troubleshooting

### JPEC says the VCF is not phased

JPEC expects phased genotypes such as `0|1`, not unphased genotypes such as `0/1`. Phase the VCF first, for example with Beagle.

### Some individuals are missing from the analysis

JPEC only analyzes individuals that are present in the VCF. If an offspring or candidate parent is absent from the VCF, it is removed from the matching set.

### I used `--score` but cannot find `scored_matrix`

The current code writes the file name as:

```text
JPEC_Sire_socred_matrix.txt
JPEC_Dam_socred_matrix.txt
```

The word `socred` is kept here because it is the current output name in the code.
